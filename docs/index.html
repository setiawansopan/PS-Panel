Get started in minutes

Four steps from a fresh VPS to a fully running panel.

01
Prepare a Ubuntu 24.04 VPS
SSH into your server as root. Minimum 1 GB RAM, 10 GB disk. Open ports 80, 443, and 8765 in your firewall.
ssh root@your-server-ip
02
Run the one-line install command
Installs and configures the complete stack automatically. Takes 5–10 minutes depending on your server's connection speed.
curl -fsSL https://raw.githubusercontent.com/setiawansopan/PS-Panel/main/install.sh | sudo bash
03
Open the panel in your browser
After install completes, navigate to your server's IP on port 8765. The install summary will show the exact URL.
http://your-server-ip:8765
04
Login with the generated password
Your credentials are displayed at the end of the install. You can always retrieve them from the credentials file.
cat /root/.pspanel_credentials
📁 Credentials file: All auto-generated passwords (panel + PostgreSQL) are saved to /root/.pspanel_credentials. Keep this file secure.

🔑 Change panel password: PANEL_PASS=newpassword pm2 restart ps-panel

Dashboard
Real-time Monitoring

The main page displaying server status in real-time. Data is automatically refreshed every 5 seconds via WebSocket.

Metric Cards
CPU · RAM · Disk · Load
Color progress bars show usage percentage. Load average is displayed for 1m / 5m / 15m intervals.
Network I/O
Download · Upload Speed
Transfer speed per second on the active interface (non-lo). 30-second bandwidth history chart.
Services
Service Status
Active/inactive status of all services (FrankenPHP, PostgreSQL, Redis, PHP-FPM, Node.js) at a glance.
Top Processes
Top 5 Processes
Processes with the highest CPU usage. Useful for detecting suspicious or resource-hungry processes.
Tip: If CPU or RAM stays at 100%, check Top Processes to identify the cause, then use the Services page to restart the relevant service.
🌐
Virtual Hosts
Domain & SSL

Manage domains served by FrankenPHP. Each virtual host generates a config file in /etc/frankenphp/sites/.

1
Add Domain
Click + Add Domain. Fill in the domain name (e.g.: myapp.com), document root (default: /var/www/myapp.com), check PHP if it's a PHP app, and SSL for HTTPS.
2
Auto SSL
When SSL is enabled, FrankenPHP (Caddy) automatically requests a certificate from Let's Encrypt. Ensure the domain points to the server IP and port 80 is open before enabling SSL.
3
Edit Environment (.env)
Click the ⚙ ENV button on a domain to edit the app's .env file directly from the browser. The previous file is automatically backed up as .env.bak.
4
Open Website
Click the domain name (displayed as a link) to open the website in a new tab. The ↗ icon indicates the domain is clickable.
Warning: Removing a vhost (Remove button) only removes the FrankenPHP configuration — files in the document root are NOT deleted.
📁
File Manager
Filesystem Access

Access the entire server filesystem directly from the browser. Starts at /var/www by default, can navigate anywhere.

→Navigate: Click a folder to enter, click the breadcrumb to go back, or type the path manually in the path bar above.
→Edit file: Click the name of a text file (PHP, JS, HTML, .env, etc.) to open the inline editor. Maximum editable file size: 2MB.
→Upload: Click the Upload button then select files from your computer. Multiple files can be uploaded at once.
→Download: Click the ↓ button on the file you want to download.
→New Folder / New File: Buttons at the top to create a new folder or file in the current directory.
→Rename / Delete: Action buttons on each file/folder row. Delete prompts for confirmation for non-empty folders.
→Chmod: Change file permissions (e.g.: 644, 755, 600) directly from the action button.
Shortcut paths: Use the quick-jump buttons (/var/www, /etc/frankenphp, /opt/ps-panel, /root) to quickly navigate to important directories.
✉️
Mail / SMTP
Email Configuration

Configure external SMTP for Laravel/PHP apps in your vhosts. Settings are saved to the app's .env file as MAIL_* variables.

1
Select Virtual Host
Choose the app domain from the dropdown. SMTP configuration will be saved to that app's .env file.
2
Choose Provider or Fill Manually
7 provider presets available: Gmail, Outlook, Yahoo, SendGrid, Mailgun, Postmark, or Custom. Click a preset to auto-fill host & port.
3
Enter Credentials
Enter the username (email/API key) and password/secret. Choose encryption: TLS (port 587), SSL (port 465), or None.
4
Test & Save
Click Send Test Email to verify the connection before saving. If successful, click Save to .env.
Gmail: Use an App Password (not your main password). Enable 2FA on your Google account, then create an App Password at myaccount.google.com/apppasswords.
▲
Deploy
Git Deployment

Deploy applications from a GitHub/GitLab repository to the server. Supports both public and private repos, plus auto-deploy via webhooks.

Step 1
SSH Deploy Key
For private repos, display the SSH public key then add it to GitHub repo → Settings → Deploy Keys. Only needs to be done once.
Step 2
Clone Repository
Clone the repo to the virtual host folder for the first time. Select the target vhost, enter the repo URL, click Clone.
Step 3
Manual Deploy
Run git pull from the browser. Laravel mode automatically runs composer, migrate, and artisan cache commands.
Step 4
Auto Deploy (Webhook)
Set up a GitHub webhook so deployment runs automatically on every push to a specific branch.
Laravel Mode: Check "Laravel App" when deploying to automatically run: composer install → php artisan migrate --force → config:cache → route:cache → view:cache.
🗄️
PostgreSQL
Database Management

Monitor and manage PostgreSQL 16 databases. Connects directly to the local cluster as the postgres user.

→Connection stats: Total active, idle, and query-running connections. Useful for detecting connection leaks.
→Database list: All databases with their sizes. Click a database name to view active connections.
→Create new database: Enter a database name and click Create. The database is immediately available without SSH.
→User management: Create new users/roles with passwords, then assign privileges to specific databases (SELECT, INSERT, UPDATE, DELETE, or ALL).
→Grant privileges: Select a user, database, and desired privilege. The "Make Owner" option grants full access — recommended for Laravel applications.
For Laravel: Create database → create user → grant ALL PRIVILEGES or check "Make Owner". Then update DB_DATABASE, DB_USERNAME, DB_PASSWORD in the app's .env via Virtual Hosts → ENV.
🔴
Redis
Cache Monitoring

Monitor Redis statistics running at 127.0.0.1:6379. Connects without authentication (Redis is bound to localhost for security).

Hit Rate
Cache Efficiency
Percentage of cache hits vs misses. Ideally >90%. If low, consider increasing maxmemory or reviewing the app's caching strategy.
Memory
Memory Usage
Used vs peak memory. When approaching maxmemory, Redis will start evicting keys according to the configured policy.
Ops/sec
Throughput
Number of commands processed per second. Useful for detecting load spikes from applications.
Keyspace
Key Count
Total keys per database (db0, db1, etc.). If keys keep growing without dropping, ensure the app sets TTL on each key.
⚙️
Services
System Services

Control system services via systemctl directly from the browser. Supports start, stop, and restart for all panel services.

→FrankenPHP — Main web server. Restart required after modifying the Caddyfile or PHP configuration.
→PHP-FPM — PHP processor. Restart if there are changes to php.ini or PHP extensions.
→PostgreSQL — Database server. Restart carefully — all active connections will be dropped.
→Redis — Cache server. In-memory data will be lost if persistence (RDB/AOF) is not configured.
→Node.js (PS Panel) — The panel itself runs via PM2. Restarting the panel will reload the latest configuration.
Warning: Stopping a service in production will cause downtime. Use Restart (not Stop) to apply configuration changes.
🔧
Settings
PHP & FrankenPHP Config

Edit PHP and FrankenPHP (Caddyfile) configuration directly from the browser. Changes require password confirmation and trigger automatic FrankenPHP restart.

→memory_limit — Per-process PHP memory limit. Default: 256M. Increase for heavy apps like Composer or image processing.
→max_execution_time — PHP script execution time limit (seconds). Default: 60. Increase for long processes like data imports.
→upload_max_filesize & post_max_size — Maximum file upload size. Both must be increased together for large file uploads to work.
→PHP Extensions — List of active extensions. Pre-installed: pgsql, redis, opcache, gd, mbstring, curl, zip, xml, and more.
→Caddyfile — Main FrankenPHP configuration. Edit carefully; a syntax error can prevent the web server from starting.
Edit flow: Click Unlock Edit → enter panel password → edit values → click Save Changes. FrankenPHP will restart automatically and the old configuration is backed up.
⬆️
Update
Self-Update

Update PS Panel to the latest version directly from the browser without needing SSH. Old files are automatically backed up before being replaced.

1
Check Update
Click Check Update. The panel compares the installed version with the latest version on GitHub. A green "new" badge appears in the sidebar automatically when a new update is available on panel load.
2
View Changelog
The list of new features and fixes in the latest version will be displayed after a successful update check.
3
Update Now
Click Update Now. The panel will download the latest files, run npm install, then restart automatically. The process usually completes in 30–60 seconds.
4
Page Auto-reloads
After the panel finishes restarting, the browser will automatically reload. Config, vhosts, databases, and app files are unaffected by the update.
Safe: The update only touches panel files (server.js, public/index.html). All vhost configs in /etc/frankenphp/sites/, PostgreSQL databases, and app .env files remain unchanged.

Frequently asked questions

Everything you need to know before getting started.

What is PS Panel?
+
PS Panel is a lightweight, self-hosted server management panel for Ubuntu 24.04 LTS. It provides a browser-based interface to monitor system resources, manage services (FrankenPHP, PHP-FPM, PostgreSQL, Redis), configure virtual hosts with SSL, and deploy applications via git — without needing SSH for routine tasks.
Is PS Panel free?
+
Yes, completely free and open-source. No subscriptions, no usage limits, no hidden costs. PS Panel is released for the open-source community. You can use it for personal projects or commercial deployments, and contribute to its development on GitHub.
Which operating systems are supported?
+
PS Panel is optimized and tested for Ubuntu 24.04 LTS. The installer may work on other Debian-based systems, but Ubuntu 24.04 is the only officially supported target. Support for additional distributions may be added in future versions.
How do I change the panel password?
+
Restart the panel with the PANEL_PASS environment variable:

PANEL_PASS=yournewpassword pm2 restart ps-panel

The current credentials are always readable from /root/.pspanel_credentials.
Is the panel secure?
+
PS Panel includes: JWT authentication (8-hour expiry), bcrypt password hashing, login rate limiting (5 attempts / 15 min), WebSocket authentication, command injection prevention on deploy inputs, and domain validation on virtual host creation. For production use, we recommend restricting port 8765 to trusted IPs via your server's firewall.
