Get started in minutes

Four steps from a fresh VPS to a fully running panel.

01
Prepare a Ubuntu 24.04 VPS
SSH into your server as root. Minimum 1 GB RAM, 10 GB disk. Open ports 80, 443, and 8765 in your firewall.
ssh root@your-server-ip
02
Run the one-line install command
Installs and configures the complete stack automatically. Takes 5–10 minutes depending on your server's connection speed.
curl -fsSL https://raw.githubusercontent.com/setiawansopan/PS-Panel/main/install.sh | sudo bash
03
Open the panel in your browser
After install completes, navigate to your server's IP on port 8765. The install summary will show the exact URL.
http://your-server-ip:8765
04
Login with the generated password
Your credentials are displayed at the end of the install. You can always retrieve them from the credentials file.
cat /root/.pspanel_credentials
📁 Credentials file: All auto-generated passwords (panel + PostgreSQL) are saved to /root/.pspanel_credentials. Keep this file secure.

🔑 Change panel password: PANEL_PASS=newpassword pm2 restart ps-panel

Dashboard
Real-time Monitoring

Halaman utama yang menampilkan kondisi server secara real-time. Data diperbarui otomatis setiap 5 detik via WebSocket.

Metric Cards
CPU · RAM · Disk · Load
Bar progress warna menunjukkan persentase penggunaan. Load average ditampilkan untuk 1m / 5m / 15m.
Network I/O
Download · Upload Speed
Kecepatan transfer per detik pada interface aktif (bukan lo). Grafik 30-detik terakhir.
Services
Status Layanan
Status aktif/tidak aktif semua layanan (FrankenPHP, PostgreSQL, Redis, PHP-FPM, Node.js) dalam satu pandangan.
Top Processes
5 Proses Teratas
Proses dengan penggunaan CPU tertinggi. Berguna untuk mendeteksi proses yang mencurigakan atau boros resource.
Tip: Jika CPU atau RAM terus 100%, cek bagian Top Processes untuk mencari proses penyebabnya, lalu gunakan halaman Services untuk restart layanan terkait.

Virtual Hosts
Domain & SSL

Kelola domain yang di-serve oleh FrankenPHP. Setiap virtual host menghasilkan file konfigurasi di /etc/frankenphp/sites/.

1
Tambah Domain
Klik tombol + Add Domain. Isi nama domain (contoh: myapp.com), document root (default: /var/www/myapp.com), centang PHP jika aplikasi PHP, dan SSL jika ingin HTTPS.
2
SSL Otomatis
Jika SSL diaktifkan, FrankenPHP (Caddy) akan otomatis request sertifikat dari Let's Encrypt. Pastikan domain sudah diarahkan ke IP server dan port 80 terbuka sebelum mengaktifkan SSL.
3
Edit Environment (.env)
Klik tombol ⚙ ENV pada domain untuk edit file .env aplikasi langsung dari browser. File lama otomatis di-backup sebagai .env.bak.
4
Buka Website
Klik nama domain (akan muncul sebagai link) untuk membuka website di tab baru. Ikon ↗ menandakan domain bisa diklik.
Perhatian: Hapus vhost (tombol Remove) hanya menghapus konfigurasi FrankenPHP — file di document root tidak ikut terhapus.
📁
File Manager
Filesystem Access

Akses seluruh filesystem server langsung dari browser. Mulai dari /var/www secara default, bisa navigasi ke mana saja.

→Navigasi: Klik folder untuk masuk, klik breadcrumb untuk kembali, atau ketik path manual di input bar atas.
→Edit file: Klik nama file teks (PHP, JS, HTML, .env, dll) untuk membuka editor inline. Maksimum ukuran file yang bisa diedit: 2MB.
→Upload: Klik tombol Upload lalu pilih file dari komputer. Bisa upload multiple file sekaligus.
→Download: Klik tombol ↓ pada file yang ingin diunduh.
→New Folder / New File: Tombol di bagian atas untuk membuat folder atau file baru di direktori saat ini.
→Rename / Delete: Tombol aksi di setiap baris file/folder. Delete meminta konfirmasi untuk folder yang tidak kosong.
→Chmod: Ubah permission file (contoh: 644, 755, 600) langsung dari tombol aksi.
Shortcut paths: Gunakan tombol quick-jump (/var/www, /etc/frankenphp, /opt/ps-panel, /root) untuk berpindah cepat ke direktori penting.
✉️
Mail / SMTP
Email Configuration

Konfigurasi SMTP eksternal untuk aplikasi Laravel/PHP di vhost. Setting disimpan ke file .env aplikasi sebagai variabel MAIL_*.

1
Pilih Virtual Host
Pilih domain aplikasi dari dropdown. Konfigurasi SMTP akan disimpan ke .env aplikasi tersebut.
2
Pilih Provider atau Isi Manual
Tersedia 7 preset provider: Gmail, Outlook, Yahoo, SendGrid, Mailgun, Postmark, atau Custom. Klik preset untuk mengisi host & port otomatis.
3
Isi Kredensial
Masukkan username (email/API key) dan password/secret. Pilih enkripsi: TLS (port 587), SSL (port 465), atau None.
4
Test & Simpan
Klik Kirim Test Email untuk memverifikasi koneksi sebelum menyimpan. Jika berhasil, klik Simpan ke .env.
Gmail: Gunakan App Password (bukan password utama). Aktifkan 2FA di akun Google, lalu buat App Password di myaccount.google.com/apppasswords.
▲
Deploy
Git Deployment

Deploy aplikasi dari repository GitHub/GitLab ke server. Mendukung repo publik maupun privat, serta auto-deploy via webhook.

Step 1
SSH Deploy Key
Untuk repo privat, tampilkan SSH public key lalu tambahkan ke GitHub repo → Settings → Deploy Keys. Cukup dilakukan sekali.
Step 2
Clone Repository
Clone repo ke folder virtual host untuk pertama kali. Pilih vhost tujuan, masukkan URL repo, klik Clone.
Step 3
Manual Deploy
Jalankan git pull dari browser. Mode Laravel otomatis menjalankan composer, migrate, dan artisan cache.
Step 4
Auto Deploy (Webhook)
Setup webhook GitHub agar deploy berjalan otomatis setiap kali ada push ke branch tertentu.
Laravel Mode: Centang "Laravel App" saat deploy untuk menjalankan otomatis: composer install → php artisan migrate --force → config:cache → route:cache → view:cache.
🗄️
PostgreSQL
Database Management

Monitor dan kelola database PostgreSQL 16. Terhubung langsung ke cluster lokal sebagai user postgres.

→Statistik koneksi: Total koneksi aktif, idle, dan yang sedang menjalankan query. Berguna untuk mendeteksi connection leak.
→Daftar database: Semua database beserta ukurannya. Klik nama database untuk melihat detail koneksi aktif.
→Buat database baru: Isi nama database lalu klik Create. Database langsung tersedia tanpa perlu SSH.
→Manajemen user: Buat user/role baru dengan password, lalu assign privilege ke database tertentu (SELECT, INSERT, UPDATE, DELETE, atau ALL).
→Grant privilege: Pilih user, database, dan privilege yang diinginkan. Opsi "Make Owner" memberikan akses penuh — direkomendasikan untuk aplikasi Laravel.
Untuk Laravel: Buat database → buat user → grant ALL PRIVILEGES atau centang "Make Owner". Kemudian update DB_DATABASE, DB_USERNAME, DB_PASSWORD di file .env aplikasi via menu Virtual Hosts → ENV.
🔴
Redis
Cache Monitoring

Monitor statistik Redis yang berjalan di 127.0.0.1:6379. Terhubung tanpa autentikasi (Redis terikat ke localhost untuk keamanan).

Hit Rate
Efisiensi Cache
Persentase cache hit vs miss. Idealnya >90%. Jika rendah, pertimbangkan menaikkan maxmemory atau review strategi cache aplikasi.
Memory
Penggunaan Memori
Memory terpakai vs peak. Jika mendekati maxmemory, Redis akan mulai mengeluarkan key (eviction) sesuai policy yang dikonfigurasi.
Ops/sec
Throughput
Jumlah command per detik yang diproses Redis. Berguna untuk mendeteksi lonjakan beban dari aplikasi.
Keyspace
Jumlah Key
Total key per database (db0, db1, dll). Jika key terus bertambah tanpa turun, pastikan aplikasi menetapkan TTL pada setiap key.
⚙️
Services
System Services

Kontrol layanan sistem via systemctl langsung dari browser. Mendukung start, stop, dan restart untuk semua layanan panel.

→FrankenPHP — Web server utama. Restart diperlukan setelah mengubah Caddyfile atau konfigurasi PHP.
→PHP-FPM — Prosesor PHP. Restart jika ada perubahan pada php.ini atau ekstensi PHP.
→PostgreSQL — Database server. Restart dengan hati-hati — semua koneksi aktif akan terputus.
→Redis — Cache server. Data in-memory akan hilang jika tidak dikonfigurasi persistensi (RDB/AOF).
→Node.js (PS Panel) — Panel itu sendiri berjalan via PM2. Restart panel akan memuat ulang konfigurasi terbaru.
Perhatian: Stop layanan di server produksi akan menyebabkan downtime. Gunakan Restart (bukan Stop) untuk menerapkan perubahan konfigurasi.
🔧
Settings
PHP & FrankenPHP Config

Edit konfigurasi PHP dan FrankenPHP (Caddyfile) langsung dari browser. Perubahan memerlukan konfirmasi password dan restart FrankenPHP otomatis.

→memory_limit — Batas memori per proses PHP. Default: 256M. Naikkan untuk aplikasi berat seperti Composer atau image processing.
→max_execution_time — Batas waktu eksekusi script PHP (detik). Default: 60. Naikkan untuk proses panjang seperti import data.
→upload_max_filesize & post_max_size — Ukuran maksimum file upload. Keduanya harus dinaikkan bersamaan agar upload besar bisa berjalan.
→Ekstensi PHP — Daftar ekstensi yang aktif. Ekstensi pre-installed: pgsql, redis, opcache, gd, mbstring, curl, zip, xml, dan lainnya.
→Caddyfile — Konfigurasi utama FrankenPHP. Edit dengan hati-hati; syntax error dapat membuat web server tidak bisa start.
Alur edit: Klik Unlock Edit → masukkan password panel → edit nilai → klik Save Changes. FrankenPHP akan restart otomatis dan konfigurasi lama dibackup.
⬆️
Update
Self-Update

Perbarui PS Panel ke versi terbaru langsung dari browser tanpa perlu SSH. File lama otomatis di-backup sebelum diperbarui.

1
Cek Update
Klik Cek Update. Panel akan membandingkan versi terpasang dengan versi terbaru di GitHub. Badge hijau "new" di sidebar muncul otomatis jika ada update baru saat panel dibuka.
2
Lihat Changelog
Daftar fitur baru dan perbaikan pada versi terbaru akan ditampilkan setelah cek update berhasil.
3
Update Sekarang
Klik Update Sekarang. Panel akan mengunduh file terbaru, menjalankan npm install, lalu restart otomatis. Proses biasanya selesai dalam 30–60 detik.
4
Halaman Reload Otomatis
Setelah panel selesai restart, browser akan otomatis reload. Konfigurasi, vhost, database, dan file aplikasi tidak terpengaruh oleh update.
Aman: Update hanya menyentuh file panel (server.js, public/index.html). Semua konfigurasi vhost di /etc/frankenphp/sites/, database PostgreSQL, dan file .env aplikasi tidak berubah.

Frequently asked questions

Everything you need to know before getting started.

What is PS Panel?
+
PS Panel is a lightweight, self-hosted server management panel for Ubuntu 24.04 LTS. It provides a browser-based interface to monitor system resources, manage services (FrankenPHP, PHP-FPM, PostgreSQL, Redis), configure virtual hosts with SSL, and deploy applications via git — without needing SSH for routine tasks.
Is PS Panel free?
+
Yes, completely free and open-source. No subscriptions, no usage limits, no hidden costs. PS Panel is released for the open-source community. You can use it for personal projects or commercial deployments, and contribute to its development on GitHub.
Which operating systems are supported?
+
PS Panel is optimized and tested for Ubuntu 24.04 LTS. The installer may work on other Debian-based systems, but Ubuntu 24.04 is the only officially supported target. Support for additional distributions may be added in future versions.
How do I change the panel password?
+
Restart the panel with the PANEL_PASS environment variable:

PANEL_PASS=yournewpassword pm2 restart ps-panel

The current credentials are always readable from /root/.pspanel_credentials.
Is the panel secure?
+
PS Panel includes: JWT authentication (8-hour expiry), bcrypt password hashing, login rate limiting (5 attempts / 15 min), WebSocket authentication, command injection prevention on deploy inputs, and domain validation on virtual host creation. For production use, we recommend restricting port 8765 to trusted IPs via your server's firewall.
