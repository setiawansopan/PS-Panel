Get started in minutes

Four steps from a fresh VPS to a fully running panel.

01
Prepare a Ubuntu 24.04 VPS
SSH into your server as root. Minimum 1 GB RAM, 10 GB disk. Open ports 80, 443, and 8765 in your firewall.
ssh root@your-server-ip
02
Run the one-line install command
Installs and configures the complete stack automatically. Takes 5–10 minutes depending on your server's connection speed.
curl -fsSL https://raw.githubusercontent.com/setiawansopan/PS-Panel/main/install.sh | sudo bash
03
Open the panel in your browser
After install completes, navigate to your server's IP on port 8765. The install summary will show the exact URL.
http://your-server-ip:8765
04
Login with the generated password
Your credentials are displayed at the end of the install. You can always retrieve them from the credentials file.
cat /root/.pspanel_credentials
📁 Credentials file: All auto-generated passwords (panel + PostgreSQL) are saved to /root/.pspanel_credentials. Keep this file secure.

🔑 Change panel password: PANEL_PASS=newpassword pm2 restart ps-panel

Frequently asked questions

Everything you need to know before getting started.

What is PS Panel?
+
PS Panel is a lightweight, self-hosted server management panel for Ubuntu 24.04 LTS. It provides a browser-based interface to monitor system resources, manage services (FrankenPHP, PHP-FPM, PostgreSQL, Redis), configure virtual hosts with SSL, and deploy applications via git — without needing SSH for routine tasks.
Is PS Panel free?
+
Yes, completely free and open-source. No subscriptions, no usage limits, no hidden costs. PS Panel is released for the open-source community. You can use it for personal projects or commercial deployments, and contribute to its development on GitHub.
Which operating systems are supported?
+
PS Panel is optimized and tested for Ubuntu 24.04 LTS. The installer may work on other Debian-based systems, but Ubuntu 24.04 is the only officially supported target. Support for additional distributions may be added in future versions.
How do I change the panel password?
+
Restart the panel with the PANEL_PASS environment variable:

PANEL_PASS=yournewpassword pm2 restart ps-panel

The current credentials are always readable from /root/.pspanel_credentials.
Is the panel secure?
+
PS Panel includes: JWT authentication (8-hour expiry), bcrypt password hashing, login rate limiting (5 attempts / 15 min), WebSocket authentication, command injection prevention on deploy inputs, and domain validation on virtual host creation. For production use, we recommend restricting port 8765 to trusted IPs via your server's firewall.
